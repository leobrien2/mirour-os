ROADMAP — TAGGING IS THE STRATEGY

Phase 1: Auto-Tag on Import (NOW)

Brand uploads CSV or POS export → fills onboarding context form → system batches products → LLM returns consistent tags → stored in product_tags.
Progress: Auto-tagging working externally. Tag-Based Product Display Mode shipped ✓. Soberish data web-scraped with auto-tags. G&Y CSV processed: 3,063 products / 6 batches (Mar 27). In active sprint for integration into mirour2 repo.
Known issues from G&Y test: Some compound tags (e.g., "leather-charm") missing split versions ("leather", "charm"). Some products have no tags (empty brackets). Kelley reviewing — not a blocker for flows.
Next: Clean G&Y CSV, integrate into product import flow, use as first production auto-tag case.

Phase 2: Learning Tags from Flows (NEXT)

AI goes back and adds new tags based on flow prompts. Tags compound over time.
