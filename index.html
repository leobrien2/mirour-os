ROADMAP — TAGGING IS THE STRATEGY

Phase 1: Auto-Tag on Import (NOW)

Brand uploads CSV or POS export → fills onboarding context form → system batches 100 products → LLM returns consistent tags → stored in product_tags. Key: prompt engineering so batch 2 doesn't invent different names than batch 1.
Progress: Tanmay web-scraped Soberish and built a tagging algo (Mar 24). Full LLM pipeline not yet in codebase — zero auto-tagging code in mirour2 repo.
Test with: Glad & Young as first real LLM case

Phase 2: Learning Tags from Flows (NEXT)

AI goes back and adds new tags based on flow prompts. "Dresses with pockets" creates a pocket tag retroactively across inventory. Tags compound over time.

Phase 3: Two-Way Ecom Sync (FUTURE — The Moat)

Mirour tags flow back to Shopify/ecom. Brands' stores become searchable + agentic-ready. In-store intelligence feeds back to ecommerce. Eng doc already exists in Google Drive. Nobody else does this.

KEY DECISIONS — MAR 26 TEAM MEETING + MAR 9 PRODUCT REVIEW
✓ Keep flows, don't go chat-only. Flows = experiential leverage, A/B testing, promotional capability. (Mar 26)

✓ Hide recommendation logic. Any/all/zone picker confuses users. Default to "any match" — hide entirely. Show tags instead. (Mar 26)

✓ New tag-based display mode. Simple filter: type a tag → see all products with that tag. For Soberish education pages. (Mar 26)

✓ Switch quiz from points to tags. Point-based system too complex for non-technical users. Tags already used for site navigation. (Mar 9)

✓ Simplify analytics. Remove average time, drop-off by step, peak hours. Focus on: total scans, completion rate, most saved items, date range comparison. (Mar 9)

✓ Mid-size brands over pop-ups. Pop-ups lack bandwidth. Enterprise (Nike, eBay, Kohl's) = 6-12 month sales cycle. Nail small-to-mid first. (Mar 20, Mar 26)

✓ Server infrastructure by May. Supabase sufficient for MVP. AWS startup credits available. Need separate dev/prod environments. Target: 10 paying clients before angel round. (Mar 20)

✗ Deprioritized: Zones, Profile Engine scoring, multi-onboarding strategies. Tags alone handle all current pilots.
